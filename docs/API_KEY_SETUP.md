# 나라장터 API 키 발급 및 설정 가이드

## 📌 개요

나라장터 입찰공고 검색을 위해서는 공공데이터포털에서 API 키를 발급받아야 합니다.

---

## 1️⃣ 공공데이터포털 가입 및 로그인

1. **공공데이터포털** 접속: https://www.data.go.kr/
2. 회원가입 (소셜 로그인 가능)
3. 로그인

---

## 2️⃣ 나라장터 API 찾기

### 검색
1. 상단 검색창에 **"나라장터 입찰공고"** 또는 **"BidPublicInfoService"** 검색
2. 검색 결과에서 다음 API 선택:
   - **나라장터 전자조달 입찰공고 목록 정보조회 서비스**
   - 또는 **BidPublicInfoService04**

### API 상세 페이지
- 제공기관: 조달청
- 서비스명: BidPublicInfoService04
- 오퍼레이션: `getBidPblancListInfoServcPPSSrch`

---

## 3️⃣ 활용신청

1. **"활용신청"** 버튼 클릭
2. 신청 정보 입력:
   ```
   활용 목적: 개인 프로젝트
   상세 기능: 나라장터 입찰공고 검색 및 제안서 자동생성 서비스
   트래픽: 하루 1,000건 이하
   ```
3. **"신청"** 버튼 클릭

### 승인 대기
- 보통 **즉시 승인** 또는 **1-2시간 내 승인**
- 승인 완료 시 이메일 알림

---

## 4️⃣ API 키 확인 ⭐ 가장 중요!

### 마이페이지에서 확인
1. 우측 상단 **"마이페이지"** 클릭
2. 좌측 메뉴 **"오픈API"** 클릭
3. 신청한 API 찾아서 **"개발계정 상세보기"** 클릭

### 인증키 복사

화면에 **2가지 키**가 표시됩니다:

```
┌──────────────────────────────────────────────────────────┐
│ 일반 인증키 (Encoding)                                    │
│ ┌────────────────────────────────────────────────────┐  │
│ │ NmME9H...%2FTYsRl...%3D%3D                         │  │
│ │              ↑ %2F = /의 인코딩                    │  │
│ │              ↑ %3D = =의 인코딩                    │  │
│ └────────────────────────────────────────────────────┘  │
├──────────────────────────────────────────────────────────┤
│ 일반 인증키 (Decoding) ⭐ 이것을 사용!                   │
│ ┌────────────────────────────────────────────────────┐  │
│ │ NmME9H.../TYsRl...==                               │  │
│ │              ↑ 실제 / 문자                         │  │
│ │              ↑ 실제 = 문자                         │  │
│ └────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────┘
```

### ✅ 어느 키를 사용할까?

**→ "일반 인증키 (Decoding)" 우측의 복사 버튼 클릭!**

- ✅ **권장**: Decoding 키 (원본 키)
- ❌ **비권장**: Encoding 키 (이미 URL 인코딩된 키)

**이유**: 현재 코드가 자동으로 URL 인코딩 처리하므로 원본 키를 사용해야 합니다.

## 5️⃣ .env.local 파일 설정

### 환경변수 파일 생성

프로젝트 루트에서:

```bash
# .env.local 파일이 없다면
cp .env.local.example .env.local
```

### API 키 입력

`.env.local` 파일을 열어서 복사한 **Decoding 키**를 붙여넣기:

```env
# 나라장터 공공데이터 API
NARA_API_KEY=여기에_복사한_Decoding_키_붙여넣기
```

### 예시

```env
# ✅ 올바른 예 (Decoding 키)
NARA_API_KEY=NmME9H620V8JEPBBQU/TYsRlEZqIRTcL1LA18PSIFtSYYyI0OGfQ/hddu3AdyUPaWGiRD0JLQ2ztlKiayRoy5A==

# ❌ 잘못된 예 (Encoding 키 - 사용하지 마세요)
NARA_API_KEY=NmME9H620V8JEPBBQU%2FTYsRlEZqIRTcL1LA18PSIFtSYYyI0OGfQ%2Fhddu3AdyUPaWGiRD0JLQ2ztlKiayRoy5A%3D%3D
```

## 🧪 테스트

### 브라우저에서 테스트

1. http://localhost:3000/dashboard 접속
2. 검색어 입력 (예: "소프트웨어")
3. "검색" 버튼 클릭
4. 결과 확인

### API 직접 테스트

```bash
curl "http://localhost:3000/api/nara/search?q=소프트웨어&limit=5"
```

**성공 응답:**
```json
{
  "success": true,
  "data": [...],
  "pagination": {...}
}
```

**실패 응답:**
```json
{
  "success": false,
  "error": "나라장터 API 오류: [03] SERVICE KEY IS NOT REGISTERED ERROR."
}
```

## ❗ 문제 해결

### 1. "SERVICE KEY IS NOT REGISTERED ERROR"

**원인**: 잘못된 API 키 또는 승인되지 않은 API

**해결**:
1. API가 승인되었는지 확인 (마이페이지 → 오픈API)
2. **Decoding 키**를 사용했는지 확인 (Encoding 키 ❌)
3. 복사할 때 앞뒤 공백이 포함되지 않았는지 확인
4. 개발 서버 재시작

### 2. "NARA_API_KEY 환경변수가 설정되지 않았습니다"

**원인**: `.env.local` 파일이 없거나 변수명이 잘못됨

**해결**:
1. `.env.local` 파일이 프로젝트 루트에 있는지 확인
2. 변수명이 정확히 `NARA_API_KEY`인지 확인 (대소문자 구분)
3. 개발 서버 재시작

### 3. "LIMIT OVER ERROR"

**원인**: API 호출 횟수 초과

**해결**:
- 공공데이터포털에서 일일 트래픽 제한 확인
- 일반적으로 하루 1,000건 제한
- 필요시 트래픽 증량 신청

### 4. 여전히 500 에러

**디버깅**:

개발 서버 터미널에서 다음 로그 확인:
```
[NARA API] 요청 URL: ...
[NARA API] 응답 코드: 00
[NARA API] 검색 성공: 156 건
```

에러가 있다면 에러 메시지 확인:
```
[NARA API] 응답 오류: { resultCode: '03', resultMsg: '...' }
```
